# AGENTS.md — AI 视频创作教程与作品集

> 本文档面向 AI Agent。请你在操作本仓库前阅读此文档，遵循其中的目录结构、命名规范和工作流程。

## 1. 项目定位

本仓库用于系统化存放 **使用 AI 进行视频创作** 的全过程资料，包括：

- **教程文档**：提示词技巧、工作流说明、工具使用经验
- **中间产物**：分镜、草稿、素材、迭代版本
- **最终成品**：成片、封面、发布文案、平台版本

它既是一个 **知识沉淀库**，也是一个 **作品归档库**。所有内容按「项目」组织，公共资源独立沉淀。

## 2. 目录结构

```
ai-video-creation/
├── README.md              # 面向人类的项目总览
├── AGENTS.md              # 本文档：面向 AI Agent 的操作指南
├── bailian-output/        # bailian-cli 默认输出目录（图片、视频等）
│
├── shared/                # 跨项目复用的公共资源
│   ├── prompts/           # 通用提示词（画面、镜头、风格、人物等）
│   ├── workflows/         # 工作流文档（Sora / Runway / Kling / ComfyUI / 剪辑等）
│   ├── templates/         # 模板文件（封面、字幕条、视频模板、剪辑预设）
│   └── tools/             # 辅助脚本、配置文件、批量处理工具
│
├── tutorials/             # 通用教程、经验总结、踩坑记录
│   └── YYYY-MM-topic.md
│
└── projects/              # 按项目组织的完整创作链路
    └── YYYY-MM-project-name/
        ├── README.md      # 项目说明、目标、使用工具、成品链接
        ├── concept.md     # 创意概念、分镜脚本、参考图、风格定义
        ├── prompts/       # 项目专属提示词
        ├── storyboards/   # 分镜、草图、参考图
        ├── assets/        # 素材（图片、音频、视频片段等）
        ├── drafts/        # 草稿、中间产物、迭代版本
        └── final/         # 最终成片、封面、发布文案
```

## 3. 命名规范

### 3.1 项目目录

统一格式：`YYYY-MM-简短英文标题`

- 示例：`2026-06-kimi-k3-promo`、`2026-07-summer-vlog`
- 全部小写，单词用连字符 `-` 连接
- 避免空格、中文、特殊符号

### 3.2 文件命名

- 图片/视频：`序号-简短描述.扩展名`
  - 示例：`01-opening-shot.png`、`02-final-cut.mp4`
- 提示词文件：`场景-用途.md` 或 `场景-用途.txt`
  - 示例：`opening-visual-prompt.md`
- 教程文档：`YYYY-MM-主题.md`
  - 示例：`2026-06-sora-workflow.md`

## 4. Agent 工作原则

### 4.1 开始前

1. 检查当前操作的对象是 **整个仓库** 还是 **某个子项目**。
2. 通用资源、教程、全局配置 → 放在仓库根目录对应目录。
3. 单个项目的内容 → 必须放到 `projects/YYYY-MM-project-name/` 下。

### 4.2 创建新项目时

1. 在 `projects/` 下按命名规范创建目录。
2. 初始化以下文件：
   - `README.md`（项目占位或简介）
   - `concept.md`（创意占位）
3. 按需创建子目录：`prompts/`、`storyboards/`、`assets/`、`drafts/`、`final/`。
4. **不要**在用户未明确说明前填充具体内容。

### 4.3 生成素材时

1. 优先使用 `bailian-cli` 生成图片/视频。
2. 默认输出目录已配置为 `bailian-output/`，生成的文件会先落在这里。
3. 生成后，将成品按项目归类到对应 `projects/.../assets/` 或 `final/` 中。
4. 不要在仓库中保留重复的大文件；必要时用链接或清单记录来源。

### 4.4 提交代码/内容时

1. 避免提交大体积二进制文件到 Git（视频、高清图片序列等）。
2. 对于大文件，记录清单（如 `assets/README.md`）而非直接入库。
3. 保持提交信息简洁清晰，中文或英文均可。

## 5. 工具配置

### 5.1 bailian-cli

- 命令：`bl`
- 默认输出目录：`bailian-output/`
- 已配置 API Key，可直接用于图片/视频生成
- 常用命令：
  - `bl image generate --prompt "..." --n 1`
  - `bl video generate --prompt "..."`
  - `bl auth status --output json`

### 5.2 其他工具

- 视频剪辑：用户本地工具（如剪映、Premiere、DaVinci Resolve）
- 脚本/自动化：可放入 `shared/tools/`

## 6. 禁止事项

- ❌ 不要把 API Key、密码等敏感信息写入仓库任何文件。
- ❌ 不要把未经用户确认的成品直接发布到公开平台。
- ❌ 不要修改 `.gitignore` 以外的仓库级配置而不告知用户。
- ❌ 不要把 bailian-output/ 中的临时文件长期堆积在根目录，应及时归档到对应项目。
