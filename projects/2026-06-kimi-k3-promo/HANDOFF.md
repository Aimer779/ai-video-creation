## 三、关键决策与经验教训

### ✅ 做得对的

1. **先处理素材再剪辑**
   - 统一裁成正方形、统一尺寸后再进剪辑流程，避免后续反复调整
   - 900×900 比输出 720×720 大，给 Ken Burns 缩放留了余量

2. **低分辨率素材果断替换**
   - 07、08 原图分辨率低于 720，硬放大会糊
   - AI 生成替代图虽然不如真实扫描，但在 0.5-0.8 秒镜头中可用

3. **先做样片再推进**
   - 先做 3 秒版快速验证节奏，再调整到 6.5 秒
   - 避免直接做完整片后发现节奏不对

4. **脚本化生成流程**
   - 把渲染逻辑抽成 `generate_montage.py`，方便第二阶段、第三阶段复用
   - 参数可调（停留时长、淡化时长、调色强度、Ken Burns 幅度）

### ⚠️ 需要注意的问题

1. **03 商周三足鼎分辨率不足**
   - 原图宽度仅 697px，处理成 900×900 是放大上去的
   - 在复杂/静态镜头中会略显软
   - **建议**：后续找一张 1200px 以上的鼎图替换

2. **08 牛顿光学图来自 AI 生成**
   - 虽然去掉了水印，但终究是生成图
   - 如果有更高清的牛顿真实手稿，建议替换

3. **各图底色/亮度差异较大**
   - 02、10 偏白；03、04 偏灰；其他偏黄
   - 全局调色只能统一一部分，更精致的做法是每张图单独做颜色匹配
   - 这在第二阶段、第三阶段也需要注意

4. **脚本时间线需同步更新**
   - 原脚本写第一阶段 0:00-0:10，但按 10 张图 × 0.83 秒算下来实际约 6.5 秒
   - 后续如果严格要 10 秒，需要增加图片数量或延长单张时间

### 💡 可复用的经验

| 经验 | 说明 |
|------|------|
| 正方形素材池 | 所有图片先统一成正方形，输出时才裁到 720×720 |
| 900×900 工作尺寸 | 比 720 大 25%，Ken Burns  zoom 到 108% 不会损失画质 |
| 交叉淡化公式 | N 张图，每张 C 帧，淡化 F 帧，总帧数 = N×C - (N-1)×F |
| 暖调调色组合 | 暖黄叠加 12% + 对比度 0.9 + 饱和度 0.92 + 纸张纹理 10% + 颗粒 |
| 生成脚本参数化 | 把渲染脚本做成命令行工具，避免每次重写 |

## 四、可复用工具

### 1. `shared/tools/generate_montage.py`

用途：将一组图片生成交叉淡化蒙太奇视频。

核心能力：
- 按文件名排序读取图片
- Ken Burns 缩放/旋转
- 交叉淡化
- 暖调调色、纸张纹理、颗粒
- 输出 H.264 MP4

常用参数：
- `--input-dir`：图片目录
- `--output`：输出 MP4 路径
- `--clip-frames`：单张停留帧数
- `--fade-frames`：交叉淡化帧数
- `--output-size`：输出分辨率
- `--start-scale` / `--end-scale`：Ken Burns 缩放范围
- `--rotation`：旋转角度
- `--warmth` / `--contrast` / `--saturation`：调色参数

### 2. 手动处理流程（当脚本不满足时）

Python + Pillow 处理单张图片的模板代码：

```python
from PIL import Image

img = Image.open("source.png").convert("RGB")
# 中心裁剪为正方形
w, h = img.size
min_dim = min(w, h)
left = (w - min_dim) // 2
top = (h - min_dim) // 2
img = img.crop((left, top, left + min_dim, top + min_dim))
# 缩放到 900×900
img = img.resize((900, 900), Image.LANCZOS)
img.save("processed.png", "PNG", optimize=True)
```
